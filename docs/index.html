Audio Concatenator

Load a folder of audio files, arrange them in any order, and join them into a single WAV with a click marker between each file.

PySide6 GUI Windows · macOS · Linux Python 3.10+ AGPL-3.0

Quick Start

  1. Install dependencies 
    pip install PySide6 pydub numpy
    You also need ffmpeg on your system PATH, or place a static binary in the bin/ folder next to the app (see FFmpeg setup).
  2. Run the app 
    python app.py
  3. Click Load Folder… and choose a folder of audio files.
  4. Arrange the files — drag rows to reorder, or use the sort controls.
  5. Choose an output path and bit depth, then click Join Files.

Simple Sort Mode

The Simple (GUI) radio button activates the standard sort controls. Choose a sort field and direction, then click Apply Sort.

Sorting only reorders what is already in the list. You can apply multiple sorts sequentially, or mix sorting with manual drag-and-drop.

Suffix Order (for Dekereke databases)

The Suffix Order panel lets you define a custom ordering for files that share a base name but differ by a suffix. This is designed for Dekereke databases, where a stimulus entry like 0001 produces files such as 0001-ynq-irt.wav, 0001-ans-irt.wav, etc.

How to use it:

  1. Type each suffix substring (or regex pattern) into the input field and click Add, or press Enter. Add one entry per suffix variant.
  2. Drag the entries to set their priority order — the file containing the first matching entry in the list will sort first within its base group.
  3. Click Apply Suffix Order.

Example — for files numbered 0001–0006, each with seven suffix variants:

Suffix list (in desired order):
  ynq-irt
  whq-irt
  ans-irt
  emphq-irt
  emph-ans-irt
  cntr-foc-irt
  cntr-top-irt

After clicking Apply Suffix Order, the file list becomes:

0001-ynq-irt.wav
0001-whq-irt.wav
0001-ans-irt.wav
0001-emphq-irt.wav
0001-emph-ans-irt.wav
0001-cntr-foc-irt.wav
0001-cntr-top-irt.wav
0002-ynq-irt.wav
0002-whq-irt.wav
…
Overlapping patterns handled automatically The app always tries the longest pattern string first when matching a filename. This means emph-ans-irt is matched before ans-irt even if ans-irt appears higher in your list — no special ordering of the entries is required for disambiguation.

Entries can be plain substrings or full Python regex patterns. If you type an invalid regex, the app auto-escapes it and treats it as a literal string.

Advanced Sort Mode (Regex)

Switch to Advanced (Regex) to sort by text extracted directly from filenames using regular expressions. This mode replaces the Simple sort entirely — use it when the GUI sort fields are not flexible enough for your naming convention.

What is a regex?

A regular expression (regex) is a pattern that describes text. The app uses Python's regex engine. You don't need to be an expert — the most useful concept here is the capture group: any text wrapped in ( ) parentheses is "captured" and used as the sort key.

Pattern:    ^(\d+)
Filename:   0042-ans-irt.wav
Captured:   0042   ← this value is used to sort the file
Tip — Ask an AI to write your regex Not sure how to write a pattern? Paste a few example filenames into Claude, ChatGPT, or another AI assistant and ask: "Write a Python regex with a capture group that extracts [the part I want to sort by] from these filenames."

The layer system

Each row in the Advanced sort panel is a sort layer. Layers work like columns in a spreadsheet sort:

  • Layer 1 (top) is the primary sort key — it has the highest priority.
  • Layer 2 breaks ties within groups that are equal under Layer 1.
  • Layer 3 breaks ties within groups equal under both Layer 1 and Layer 2, and so on.

Use ▲ / ▼ to reorder layers. Click + Add Layer to add a new one. The minimum is one layer.

Layer 1 ^(\d+) Numeric · Ascending
Layer 2 -(\w+-irt)\. Natural text · Ascending

The two-layer example above replicates the Suffix Order result: Layer 1 groups files by their leading number; Layer 2 sub-sorts them by suffix.

Controls per layer

ControlWhat it does
▲ / ▼ Move the layer up (higher priority) or down (lower priority).
Pattern A Python regex containing at least one capture group ( ). A ✓ appears when the pattern is valid; ✗ appears (with error tooltip) when it is not.
Grp Which capture group to sort by. Group 1 is the first ( ), group 2 is the second, etc. Default: 1.
Sort as How to interpret the captured text (see table below).
Ascending / Descending Sort direction for this layer.
✕ Remove this layer (minimum one layer always remains).

Sort-as modes

ModeBehaviourExample
Natural text String sort, but runs of digits are compared numerically. Recommended for most filenames. track2 < track10
Numeric Parse the captured text as a number (float). Use when the capture group contains only digits or a decimal value. 120.5 < 130
Alphabetical Plain case-insensitive A–Z string comparison. Digits sort as characters, not numbers. track10 < track2

Unmatched files

If a file's name does not match a layer's pattern, that file is sorted after all files that did match — but it still keeps its position relative to other unmatched files, and is still sorted correctly by any higher-priority layer it did match.

Regex Quick Reference

These are the building blocks you'll use most often. Every example below is a complete pattern ready to paste into the Pattern field.

PatternCapturesExample filename → captured
^(\d+) Leading digits at the start of the filename 0042-ans-irt.wav → 0042
-(\w+-irt)\. A hyphen-separated suffix ending in -irt 0001-ynq-irt.wav → ynq-irt
_(\d{4}-\d{2}-\d{2}) A date in YYYY-MM-DD format preceded by an underscore rec_2024-03-15.wav → 2024-03-15
(\d{2,3})bpm A 2–3-digit BPM value loop_128bpm_A.wav → 128
_([A-G][#b]?)_ A musical key between underscores chord_Eb_maj.wav → Eb
(\d+)$ Trailing digits at the end of the stem (before the extension) take007.wav → 007
-(spkr\d+)- A speaker ID like spkr03 item01-spkr03-cond2.wav → spkr03

Testing your pattern The ✓ / ✗ indicator only checks that the pattern is syntactically valid — it does not confirm it matches your filenames. Always click Apply Regex Sort and inspect the resulting file order before clicking Join Files.

Worked Example — Dekereke Database

Suppose your folder contains files named:

0001-ans-irt.wav      0002-ans-irt.wav      0003-ans-irt.wav
0001-ynq-irt.wav      0002-ynq-irt.wav      0003-ynq-irt.wav
0001-cntr-foc-irt.wav 0002-cntr-foc-irt.wav 0003-cntr-foc-irt.wav

You want the output order to be: all variants of 0001 first (ynq, ans, cntr-foc), then all of 0002, then 0003.

Option A — Suffix Order panel (Simple mode, recommended)

  1. Open Simple (GUI) mode.
  2. In the Suffix Order box, add entries in this order: 
    ynq-irt
ans-irt
cntr-foc-irt
  3. Click Apply Suffix Order.

Option B — Advanced (Regex) mode

  1. Open Advanced (Regex) mode.
  2. Set up two layers:
    Layer 1 ^(\d+) Numeric · Ascending — groups by base number
    Layer 2 -(\w+-irt)\. Alphabetical · Ascending — sub-sorts suffix

    Note: Alphabetical gives ans → cntr-foc → ynq. To get a custom suffix order instead, use Option A.

  3. Click Apply Regex Sort.

Bit Depth

SettingSample widthUse case
16-bit (default) 2 bytes / sample CD quality, broadest compatibility, smallest file size
24-bit 3 bytes / sample Studio / archival quality
32-bit 4 bytes / sample Post-production, maximum headroom before mixdown

All input files are normalised to 44 100 Hz mono at the chosen bit depth before being joined. The click marker between each file is a 5 ms 2 kHz sine burst with 500 ms of silence on each side.

FFmpeg Setup

The app uses ffmpeg (via pydub) to decode non-WAV formats. It looks in two places, in order:

  1. Bundled binary — bin/ffmpeg (or bin/ffmpeg.exe on Windows) placed next to app.py.
  2. System PATH — any ffmpeg installed system-wide (e.g. via brew install ffmpeg on macOS or the installer on Windows).

If neither is found, the app shows an error dialog at startup. See bin/README.md in the repository for download links and the PyInstaller .spec snippet for bundling.

FAQ

What audio formats are supported as input?

WAV, MP3, FLAC, AIFF/AIF, OGG, M4A, MP4. Any format pydub/ffmpeg can decode will work. The output is always WAV.

Can I manually reorder files after applying a sort?

Yes. Sort operations only rearrange the list — you can drag items freely before or after applying any sort. The list order at the moment you click Join Files is what gets used.

What if a file doesn't match any suffix pattern?

In Suffix Order mode, unmatched files sort after all matched files within their base group, using natural filename order. In Advanced Regex mode, unmatched files sort after matched files for that layer.

My regex is marked ✓ but the sort looks wrong. Why?

The ✓ only confirms the pattern is syntactically valid, not that it matches your filenames. Check that your capture group ( ) is in the right place and that the Grp number matches the group you intend. You can test patterns at regex101.com — choose "Python" flavour.

Can I build a standalone portable app?

Yes, via PyInstaller:

pip install pyinstaller
pyinstaller --onefile app.py

Drop ffmpeg into bin/ before packaging to bundle it. See bin/README.md for the full .spec snippet.

The join is slow for large folders. Is that normal?

Decoding, resampling, and writing audio is CPU-bound and scales with total audio duration. The join runs on a background thread so the UI stays responsive. A folder of 50 short WAV clips should finish in a few seconds; long MP3s decoded through ffmpeg take longer.
